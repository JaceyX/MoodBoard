# 豆包画板 · 多图融合（doubao-moodboard）

采集网页上的参考图，给每张图标注「要保留的内容」（风格 / 配色 / 某个元素…），再用豆包 **Seedream** 的多图参考能力把它们融合成一张全新的原画。

适合画原画前的找图工作流：图 A 的风格、图 B 的配色、图 C 的某个元素，整合到一起出图。

> Manifest V3 浏览器扩展（Chrome / Edge），侧边栏交互。当前版本 **v0.3.0**。

---

## 核心思路

不是把图拼起来，而是**把多张图当参考、配一段位置化提示词**喂给模型：

- 单次最多 **10 张**参考图，提示词里用「图1 保留风格、图2 保留配色、图3 保留某元素」的方式按位置引用；
- 生成时设 `sequential_image_generation: "disabled"`，拿**单张**融合结果；
- 接口落点：`POST https://ark.cn-beijing.volces.com/api/v3/images/generations`，模型 `doubao-seedream-4-0-250828`。

参考图在提交前会被统一**抓取 → 解码 → 缩放到长边 1536 → 转 base64**（`data:image/jpeg;base64,…`）再传入 `image` 数组，而不是直接用网页原始 URL。这样规避了 `invalid url`、防盗链、`webp` 等一系列问题。

---

## 功能

- **右键采集**：任意网页图片上右键「➕ 加入豆包画板」，侧边栏自动弹出。
- **逐图标注**：每张参考图选一个保留维度（整体风格 / 配色 / 线条 / 构图 / 光影 / 主体 / 指定元素 / 材质 / 自定义 / 以这张为基础）+ 可选补充说明。
- **拖拽排序**：卡片左上角 `⠿` 手柄拖动调整顺序，编号「图1/图2…」实时重排，直接决定提示词里的引用顺序。
- **最终提示词可见可改**：底部可展开查看自动拼接出的完整提示词；手动编辑后以你的为准（标记「已手动编辑」），可一键「恢复自动生成」。
- **多图融合生成**：1K / 2K / 4K 可选，水印可在设置里开关。
- **迭代闭环**：生成结果点「在此基础上改」，把这张结果作为「基础」加回画板，改一句描述再生成 = 图生图迭代。同一任务多次生成会累积成缩略图带，可切换查看、单独下载。
- **多任务 + 历史**：「＋」新建任务，「🕘 历史」查看 / 打开 / 删除过往任务。**结果图以 base64 落地**，因此不受方舟图片 URL 约 24 小时失效的影响，历史里长期可看可下。

## 安装

1. 下载并解压本项目。
2. 浏览器打开 `chrome://extensions`（Edge 为 `edge://extensions`）。
3. 打开右上角「**开发者模式**」。
4. 点「**加载已解压的扩展程序**」，选解压出的 `doubao-moodboard` 文件夹。

## 配置

点工具栏图标 →「⚙ 设置」：

| 项 | 说明 |
| --- | --- |
| **API Key** | 火山方舟的 `ARK_API_KEY`，在[火山方舟控制台 → API Key 管理](https://console.volcengine.com/ark)获取，并在「开通管理」里开通 Seedream（图片生成）模型。 |
| **模型** | 默认 `doubao-seedream-4-0-250828`。账号开通的是 4.5 / 5.0 就填对应模型 ID 或推理接入点 ID（`ep-…`）。 |
| **水印** | 是否给输出加水印。 |

> 凭据只保存在本机浏览器（`chrome.storage.local`），不会上传。

## 使用

1. 在网页图片上**右键 →「➕ 加入豆包画板」**。
2. 侧边栏里给每张图选保留维度（"指定元素 / 自定义"时务必在说明里写清楚）。
3. 需要的话拖动 `⠿` 调整顺序。
4. 底部写一句「最终想要的画面」，可展开「最终提示词」微调。
5. 点「**融合生成**」。
6. 想继续改 → 点「在此基础上改」，调描述后再生成。
7. 「＋」开新任务，「🕘 历史」回看以前的。

## 文件结构

```
doubao-moodboard/
├── manifest.json      # MV3 配置（权限、侧边栏、右键菜单）
├── background.js      # service worker：右键菜单、写入当前任务、打开侧边栏、角标
├── store.js           # 数据层：当前任务 / 历史索引 / 结果持久化
├── sidepanel.html     # 侧边栏：任务视图 + 历史视图
├── sidepanel.css      # 侧边栏样式
├── sidepanel.js       # 侧边栏逻辑：画板 CRUD、拖拽、提示词、生成、迭代、历史
├── options.html       # 设置页
├── options.js
└── icons/             # 16 / 48 / 128 图标
```

### 本地存储结构

| 键 | 内容 |
| --- | --- |
| `currentId` | 当前任务 id |
| `task:<id>` | 单个任务：`{ board[], prompt, promptOverride, size, results[] }` |
| `result:<rid>` | 一张结果图的完整 base64（单独存，避免任务对象过大） |
| `index` | 轻量历史列表（标题、计数、缩略图等） |
| `settings` | API Key、模型、水印 |

为存历史结果图，manifest 声明了 `unlimitedStorage` 权限。

## 已知限制

- **`blob:` / 懒加载图读不出来**：这类图的字节在侧边栏上下文里 `fetch` 不到，会明确提示是哪张。临时办法：在原网页右键「在新标签页打开图片」拿到真实地址再加入。根治方案是走 content script 在网页上下文里读字节（见下方路线）。
- **拖拽仅桌面端**：用的是 HTML5 drag，触屏不生效。
- **API Key 存在本地**：个人自用没问题；要分发给他人需把 Key 挪到服务端。
- **参考图缩略图只存原始网址**：很久以后若原图站点失效，历史里的参考缩略图可能加载不出——但**结果图不受影响**（已 base64 落地）。

## 后续可做

- **FastAPI 代理**：Key 上服务端 + 用火山 TOS 托管图片 + SSE 流式进度（也能顺带解决 `blob`/防盗链图）。
- 参考图缩略图一并 base64 落地，历史长期不裂。
- 生成的取消 / 超时（`AbortController`）、429 限流自动重试。
- 历史结果数量上限或「清理旧结果」。
